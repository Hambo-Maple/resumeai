## 1. 整体路线图

简历助手智能体的完整目标是成为“岗位定制型简历内容教练”，围绕具体公司、具体岗位和用户真实经历，完成简历生成、优化、评估和后续面试训练衔接。

### 1.1 阶段一：MVP 可用闭环

目标：让真实学生能完成一次岗位定制简历生成。

核心能力：

- 输入目标公司、目标岗位和 JD。
- 分析岗位核心能力和关键词。
- 引导用户补充经历素材。
- 生成中文简历核心内容。
- 对简历进行评分和修改建议。

### 1.2 阶段二：简历优化增强

目标：让已有简历用户可以获得更稳定、更专业的优化结果。

核心能力：

- 上传或粘贴已有简历。
- 按目标岗位重排和改写内容。
- 支持多版本对比。
- 支持单段经历反复打磨。
- 支持中英文简历转换与润色。

### 1.3 阶段三：公司与岗位信息增强

目标：让简历内容更贴合具体公司和业务背景。

核心能力：

- 公司公开信息整理。
- 岗位与公司业务线关联分析。
- 公司关键词融入简历表达。
- 支持目标公司研究摘要。

### 1.4 阶段四：导出与版本管理

目标：让用户能长期维护自己的求职材料。

核心能力：

- Word / PDF 导出。
- 简历版本历史。
- 不同岗位简历管理。
- 用户经历素材库。
- Prompt、分析过程和生成结果沉淀。

### 1.5 阶段五：与面试训练联动

目标：把简历助手与面试模拟、能力评估形成闭环。

核心能力：

- 根据简历生成面试追问。
- 根据面试反馈反向优化简历。
- 根据岗位能力缺口推荐专项训练。
- 根据用户能力和意愿推荐岗位方向。

## 2. MVP 范围

MVP 只做最小可用闭环：

```text
输入目标公司/岗位/JD
-> 分析岗位能力
-> 采集用户经历
-> 生成简历内容
-> 给出评分和修改建议
```

### 2.1 MVP 必做

- 目标公司、岗位、JD 输入。
- 岗位能力分析。
- 用户经历采集。
- 经历结构化。
- 中文简历内容生成。
- 简历评分。
- 修改建议输出。
- 保存关键输入和输出。

### 2.2 MVP 暂不做

- 精美 PDF 模板。
- 完整在线简历编辑器。
- 招聘网站接口。
- 公司信息自动爬取。
- 多用户协作。
- 完整岗位推荐系统。
- 深度英文简历能力。

## 3. MVP 用户流程

### 3.1 新建简历任务

```text
用户进入简历助手
-> 选择“生成岗位定制简历”
-> 输入目标公司、目标岗位、岗位 JD
-> 点击开始分析
```

### 3.2 岗位能力分析

```text
系统分析 JD
-> 输出岗位核心能力
-> 输出关键词
-> 输出简历应突出方向
-> 用户确认或调整能力重点
```

### 3.3 经历素材采集

```text
系统询问用户已有经历
-> 用户输入项目、实习、课程、竞赛等经历
-> 系统追问角色、行动、结果、数据
-> 系统整理为结构化经历卡片
```

### 3.4 简历生成

```text
系统根据岗位能力和经历素材
-> 生成简历核心内容
-> 包括个人简介、经历描述、技能关键词
-> 用户查看生成结果
```

### 3.5 评分与优化

```text
系统评估简历
-> 输出岗位匹配度评分
-> 标注优势和短板
-> 给出修改建议
-> 用户选择继续优化或保存结果
```

## 4. 页面拆解

### 4.1 页面一：任务创建页

用途：收集目标公司、岗位和 JD。

主要字段：

- 目标公司名称。
- 目标岗位名称。
- 岗位 JD 文本框。
- 用户身份，可选，例如本科生、研究生、转专业、实习求职。

主要操作：

- 开始分析。
- 使用无 JD 模式。

验收标准：

- 用户不需要开发者解释，就能知道先输入公司和岗位。
- 没有 JD 时，仍可继续进入通用岗位分析。

### 4.2 页面二：岗位分析页

用途：展示系统对岗位的理解。

展示内容：

- 岗位核心能力。
- 能力重要程度。
- 岗位关键词。
- 简历内容建议突出方向。

主要操作：

- 确认分析结果。
- 手动调整能力重点。
- 进入经历采集。

验收标准：

- 用户能一眼看出这个岗位需要什么能力。
- 用户能理解后续简历为什么要围绕这些能力展开。

### 4.3 页面三：经历采集页

用途：引导用户补充可写经历。

展示内容：

- 对话式追问区域。
- 已整理的经历卡片。
- 每段经历对应的能力标签。

主要操作：

- 添加经历。
- 回答追问。
- 编辑经历卡片。
- 跳过当前问题。
- 进入简历生成。

验收标准：

- 用户即使没有完整简历，也能通过提问补充素材。
- 系统不会虚构用户没有提供的信息。

### 4.4 页面四：简历生成与评估页

用途：展示简历生成结果和优化建议。

展示内容：

- 个人简介。
- 经历描述。
- 技能关键词。
- 岗位匹配度评分。
- 优势分析。
- 短板分析。
- 修改建议。

主要操作：

- 重新生成。
- 按建议优化。
- 保存当前版本。
- 复制简历内容。

验收标准：

- 输出内容能直接作为简历初稿使用。
- 反馈具体，不只是泛泛建议。

## 5. API 拆解

### 5.1 POST /api/resume/analyze-job

用途：分析目标岗位。

请求参数：

```json
{
  "company": "字节跳动",
  "position": "产品运营实习生",
  "jobDescription": "岗位 JD 文本，可为空"
}
```

响应参数：

```json
{
  "jobTargetId": "job_001",
  "source": "jd | general_model",
  "coreAbilities": [
    {
      "name": "用户洞察",
      "importance": 5,
      "description": "能够理解用户需求并转化为运营动作"
    }
  ],
  "keywords": ["用户增长", "数据分析", "活动运营"],
  "resumeFocus": ["突出用户分析经历", "补充数据结果"]
}
```

### 5.2 POST /api/resume/structure-experience

用途：将用户输入的原始经历整理成结构化经历。

请求参数：

```json
{
  "jobTargetId": "job_001",
  "rawExperience": "我做过一个校园活动运营项目..."
}
```

响应参数：

```json
{
  "experienceId": "exp_001",
  "type": "project",
  "title": "校园活动运营项目",
  "role": "活动策划与执行",
  "actions": ["设计活动流程", "撰写推文", "协调社群传播"],
  "results": ["获得 300+ 报名"],
  "matchedAbilities": ["活动运营", "沟通协作"],
  "missingInfoQuestions": ["是否有转化率、阅读量或参与率数据？"]
}
```

### 5.3 POST /api/resume/generate

用途：生成岗位定制简历内容。

请求参数：

```json
{
  "jobTargetId": "job_001",
  "experienceIds": ["exp_001", "exp_002"],
  "language": "zh"
}
```

响应参数：

```json
{
  "resumeDraftId": "draft_001",
  "summary": "具备校园活动运营与内容策划经验...",
  "sections": [
    {
      "title": "项目经历",
      "items": [
        {
          "title": "校园活动运营项目",
          "bullets": [
            "负责校园活动策划与社群传播，设计活动流程并协调内容发布，累计吸引 300+ 名学生报名参与。"
          ],
          "abilityTags": ["活动运营", "沟通协作"]
        }
      ]
    }
  ],
  "skills": ["活动策划", "数据分析", "内容运营"]
}
```

### 5.4 POST /api/resume/evaluate

用途：评估简历与岗位的匹配度。

请求参数：

```json
{
  "jobTargetId": "job_001",
  "resumeDraftId": "draft_001"
}
```

响应参数：

```json
{
  "evaluationId": "eval_001",
  "overallScore": 78,
  "dimensions": [
    {
      "name": "岗位匹配度",
      "score": 80,
      "comment": "已体现活动运营和沟通协作，但用户洞察证据不足。"
    }
  ],
  "strengths": ["有活动执行结果", "表达较清晰"],
  "weaknesses": ["缺少数据分析经历", "岗位关键词覆盖不足"],
  "suggestions": ["补充一次基于数据复盘优化活动的经历"]
}
```

### 5.5 POST /api/resume/rewrite

用途：根据反馈改写简历内容。

请求参数：

```json
{
  "resumeDraftId": "draft_001",
  "instruction": "突出数据分析能力，语言更简洁"
}
```

响应参数：

```json
{
  "resumeDraftId": "draft_002",
  "changes": ["强化数据分析表达", "压缩冗余描述"],
  "content": {}
}
```

## 6. 数据结构

## 7. Prompt 拆解

### 7.1 JD 分析 Prompt

目标：从 JD 或岗位名称中提取能力、关键词和简历重点。

输入：

- 公司。
- 岗位。
- JD。

输出：

- 核心能力。
- 关键词。
- 简历突出方向。
- 证据建议。

约束：

- 没有 JD 时，标注为通用岗位模型。
- 不要编造具体公司内部信息。

### 7.2 经历追问 Prompt

目标：根据岗位能力和用户原始经历，追问缺失细节。

输入：

- 岗位能力。
- 用户原始经历。

输出：

- 3-5 个追问问题。
- 优先追问角色、行动、结果、数据。

约束：

- 问题要具体。
- 不要一次问太多。
- 不要要求用户提供不存在的信息。

### 7.3 经历结构化 Prompt

目标：将用户输入整理为结构化经历卡片。

输入：

- 原始经历文本。
- 追问回答。
- 岗位能力。

输出：

- 经历类型。
- 标题。
- 角色。
- 行动。
- 结果。
- 可量化信息。
- 匹配能力。
- 缺失信息。

约束：

- 用户没有提供的数据不能虚构。
- 不确定信息放入 missingInfoQuestions。

### 7.4 简历生成 Prompt

目标：生成岗位定制简历内容。

输入：

- 岗位能力。
- 关键词。
- 结构化经历。

输出：

- 个人简介。
- 经历 bullet point。
- 技能关键词。

约束：

- 内容必须基于用户经历。
- 优先突出岗位匹配能力。
- 语言简洁、具体、结果导向。

### 7.5 简历评分 Prompt

目标：评估当前简历与岗位的匹配程度。

输入：

- 岗位能力。
- 简历草稿。

输出：

- 总分。
- 分维度评分。
- 优势。
- 短板。
- 修改建议。

约束：

- 反馈要具体到内容层面。
- 不要只输出泛泛建议。

### 7.6 简历改写 Prompt

目标：根据用户指令或评分反馈改写简历。

输入：

- 原简历内容。
- 目标岗位。
- 改写指令。

输出：

- 改写后版本。
- 改动说明。

约束：

- 不新增未经用户确认的事实。
- 保留原始经历真实性。

- 新建简历任务页面。
- 岗位分析接口。
- 岗位能力展示。

- 简历生成接口。
- 简历评估接口。
- 结果展示页面。
- 基础数据保存。

### P1：建议完成

- 简历改写接口。
- 简历内容复制。

### P2：后续完成

- 公司信息增强。

- 与面试模拟模块联动。

## 9. 测试样例

### 样例一：产品运营实习生

输入：

- 公司：字节跳动。
- 岗位：产品运营实习生。
- 经历：校园活动运营、公众号内容策划、用户问卷调研。

重点验证：

- 是否突出用户洞察、活动运营、数据意识。
- 是否避免空泛表达。

### 样例二：数据分析实习生

输入：

- 公司：互联网公司。
- 岗位：数据分析实习生。
- 经历：课程数据分析项目、Python 数据清洗、可视化报告。

重点验证：

- 是否突出 SQL / Python / 数据分析方法。
- 是否追问数据规模、指标、分析结论。

输入：

- 公司：软件科技公司。

重点验证：

- 是否突出技术栈、系统设计、接口开发、性能或稳定性。
- 是否追问项目规模和具体职责。

### 样例四：AI 产品经理助理

- 公司：AI 创业公司。

重点验证：

- 是否突出 AI 产品理解、需求分析、原型表达和跨团队沟通。

### 样例五：新媒体运营实习生

重点验证：

- 是否突出内容策划、传播数据、用户互动和转化意识。

## 10. 首版验收清单

- 可以完成一次从岗位输入到简历生成的完整流程。
- 岗位分析结果结构化展示。
- 经历采集能追问缺失信息。
- 简历内容与目标岗位相关，而不是通用模板。
- 评分反馈能指出具体优势和短板。
- 系统不编造用户未提供的经历和数据。
- 关键输入、输出和 Prompt 版本可以保存。
- 页面默认面向真实用户，不展示调试信息。
